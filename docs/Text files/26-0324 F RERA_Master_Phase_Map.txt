Phase 10B – Superadmin Billing Bypass & Frontend Authentication Hardening
Status: Completed
Date: March 23, 2026

Problem
Superadmin user receiving teaser/locked report on live production site
despite backend bypass logic having been committed. Root cause: interview
API calls in apiClient.js were sending auth: false, meaning no JWT token
was attached. Backend treated all interview requests as anonymous and
returned preview: true unconditionally.

Backend changes:
– user_has_admin_bypass(user) added to billing/services.py
– has_unlimited_report_access() routes through bypass check
– billing/report_access.py short-circuits for superusers
– reports/views.py: admin bypass before subscription/credit checks
– billing_type = "admin" set when superuser bypass applies
– timezone.now() replaces datetime.utcnow()

Frontend changes:
– apiClient.js: auth: true on all interview endpoints
  (startInterview, getInterview, saveInterview, submitInterview)
– InterviewPage.jsx: stale rera_interview_id auto-load removed
– ReportView.jsx: loadReport(force) guard updated; force-refetch
  after inline login

Tests added:
– tests/test_superuser_billing.py (3 regression tests, all passing)
– Backend: 17 passed | Frontend: 10 passed

Commits:
– 9eb2a97  Backend billing bypass + regression tests
– 66ae23d  Frontend auth fix + stale draft removal + ReportView guard

Result:
Full report confirmed rendering in incognito browser as superadmin.

Phase 10C – Vercel SPA Routing Fix
Status: Completed
Date: March 23, 2026

Problem
Direct navigation to any React Router route (e.g. /login, /new, /report/:id)
returned 404: NOT_FOUND from Vercel CDN. No SPA fallback rewrite existed.

Fix:
– Created frontend/vercel.json with SPA rewrite:
    { "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }] }
– All direct URL access and page refreshes now correctly serve
  index.html and allow React Router to handle routing client-side.

Commit:
– 8f90af4  add vercel rewrite for spa routes

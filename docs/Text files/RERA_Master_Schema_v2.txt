Database Configuration (Phase 6 - Staging Infrastructure)

The application now supports both traditional DB_* environment variables and modern DATABASE_URL format:

TRADITIONAL (separate variables):
DB_NAME=rera_staging
DB_USER=rera_staging
DB_PASSWORD=<password>
DB_HOST=localhost
DB_PORT=5432
DB_SSLMODE=require

RAILWAY DATABASE_URL (preferred for hosted PostgreSQL):
DATABASE_URL=postgresql://user:password@host:port/dbname
DB_SSLMODE=require

Parsing Logic:
? If DATABASE_URL is set, it takes precedence
? PostgreSQL connection strings automatically parsed
? SSL mode configurable via DB_SSLMODE environment variable
? Backward compatible with existing DB_* configuration

This enables seamless integration with hosting services like Railway,
Render, and other managed PostgreSQL providers that supply DATABASE_URL.

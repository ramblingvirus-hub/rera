PHASE 11C - EMAIL DELIVERY HARDENING + GUNICORN RUNTIME
Date: March 30, 2026
Status: COMPLETE
---------------------------------------------------

Problem

Contact submissions returned success in the UI, but delivery to the admin
inbox was failing in production.

Root Cause

Railway environment blocked outbound SMTP connectivity. Traditional Django
SMTP email settings could not reliably deliver from production.

Changes Delivered

1) Email provider migration
   - Replaced SMTP send flow with Resend HTTP API integration
   - Added RESEND_API_KEY and RESEND_FROM_EMAIL support
   - Kept CONTACT_ADMIN_EMAIL as destination inbox

2) Non-blocking delivery behavior
   - Email sending moved to a background thread so contact submission does
     not stall on network timeouts
   - Failures are logged without breaking the user-facing success response

3) Runtime hardening
   - Added Gunicorn as production WSGI server
   - Added Procfile and railway.toml startup control
   - Confirmed Railway starts Gunicorn workers instead of dev server

4) UX refinement
   - Added auto-dismiss and manual close behavior for contact success/error
     alerts in the frontend

Production Outcome
   - Contact emails now deliver through Resend
   - Backend starts under Gunicorn in production

PHASE 10B – Superadmin Billing Bypass & Frontend Auth Hardening
Date: March 23, 2026
Status: COMPLETE
---------------------------------------------------

Root Cause Investigation

After backend billing bypass was committed (billing/services.py,
billing/report_access.py, reports/views.py – commit 9eb2a97),
the superadmin was still receiving a teaser/preview report on the live site.

Root cause traced to frontend: all interview API calls in apiClient.js
were sending `auth: false`. This meant no JWT token was attached to
requests. On the backend, request.user.is_authenticated evaluated to False,
causing the SubmitInterviewView to return preview: true for every evaluation
regardless of user identity.

Backend billing bypass was correct. Frontend was bypassing it entirely by
presenting as anonymous.

1. billing/services.py
   – Added user_has_admin_bypass(user): returns True for is_superuser
   – Added has_unlimited_report_access(user): routes through bypass check
   – Updated has_sufficient_credit() to short-circuit for superusers
   – Updated enforce_report_access() to return "admin" for superusers

2. billing/report_access.py
   – Imported user_has_admin_bypass from services
   – can_access_full_report() and user_has_sufficient_credits() now
     return early for superusers

3. reports/views.py
   – EvaluateProjectView checks user_has_admin_bypass() before
     subscription or credit enforcement
   – Sets billing_type = "admin" when bypassed
   – Replaced datetime.utcnow() with timezone.now() (naive datetime fix)

4. frontend/src/api/apiClient.js
   – startInterview, getInterview, saveInterview, submitInterview:
     auth: false → auth: true
   – JWT token now included in all interview API calls

5. frontend/src/pages/InterviewPage.jsx

PHASE 10C – Vercel SPA Routing Fix
Date: March 23, 2026
Status: COMPLETE
---------------------------------------------------

Problem

During browser cache-bust testing, navigating directly to
rera-five.vercel.app/login returned:
    404: NOT_FOUND

Vercel CDN was not configured to fall back to index.html for client-side
React Router routes. Any direct URL access or page refresh on a non-root
route produced a hard 404.

Fix

Created frontend/vercel.json with SPA rewrite rule:

{
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ]
}

This instructs Vercel to serve index.html for all routes, allowing React
Router to handle the path client-side. Standard requirement for any Vite/
React Router SPA deployment on Vercel.

PHASE 10D – Interview Enter Key Navigation
Date: March 23, 2026
Status: COMPLETE
---------------------------------------------------

Feature Request

Allow the Enter key to advance to the next question during the interview
flow, in addition to the existing Next → button.

Implementation

frontend/src/pages/InterviewPage.jsx

– Extracted inline onClick logic into handleNext() function (shared by
  button and keyboard listener)
– Added useEffect with window keydown listener:
    if (e.key !== "Enter") return
    if (document.activeElement?.tagName === "SELECT") return
    handleNext()
– <select> guard prevents Enter from hijacking native dropdown behavior
– Listener is cleaned up and re-registered on currentIndex change

Behavioral Coverage
– Text input questions (type: "text"): Enter advances
– Radio card questions: Enter advances after selection
– Dropdown questions (type: "select"): Enter opens/closes native dropdown
  without advancing (guarded by SELECT tag check)
– Last question reached: Enter opens Review screen instead of advancing

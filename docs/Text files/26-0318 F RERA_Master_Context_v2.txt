System Purpose
RERA is a structured real?estate risk evaluation engine designed to assess development projects based on regulatory, legal, and environmental indicators. The system produces standardized risk reports while enforcing strict billing integrity and audit traceability.
Core Architecture
� Django backend with PostgreSQL data layer
� Immutable credit ledger for billing
� JWT authentication with short?lived access tokens
� Idempotent evaluation endpoint
� Structured audit logging system
� Request correlation via request_id
Billing Model
The system operates on a credit?based billing model. Purchases create positive ledger entries. Each evaluation deducts one credit through a usage entry. Ledger records are append?only and cannot be modified or deleted in production environments.
Financial Integrity Guarantees
� No duplicate charges under request retries
� Idempotency?Key required for evaluation requests
� Replay requests return stored responses
� Payload conflicts produce HTTP 409
� Zero balance produces HTTP 402 Payment Required
Audit Architecture
Audit events capture all lifecycle stages including:

Applied inside transaction.atomic() block.

Effect:

Purchase row is locked during deduction, preventing concurrent
requests from reading the same balance simultaneously.

Guarantee:

� No negative credit balance possible
� Ledger integrity preserved under concurrency
� Financial correctness maintained
